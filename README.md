# Open Brain вҖ” Today

Chrome new tab extension that turns every new tab into a focused dashboard: top 3 AI-prioritized tasks from your personal memory system ([Open Brain](https://github.com/izomba-lang/open-brain)), with a daily strategic brief generated by Claude.

![Screenshot](https://via.placeholder.com/1400x800.png?text=Focus+View+Screenshot)

## What it does

Every time you open a new tab, you see:

1. **A short greeting** вҖ” "Р”РҫРұСҖСӢР№ РҙРөРҪСҢ, РҳР»СҢСҸ"
2. **A strategic focus line** вҖ” one sentence from Claude: what's most important right now and why, based on ALL your context (tasks, notes, people, projects)
3. **"РҹРҫРҙСҖРҫРұРҪРөРө"** вҖ” expands into 3-5 specific advice points + warnings about risks/overdue items
4. **Top 3 tasks** вҖ” AI-prioritized, with short "verb + essence" titles (max 6 words), deadline badges, area/topic tags
5. **Quick capture bar** вҖ” type a task, hit Enter вҶ’ saved to Open Brain via MCP
6. **Explore view** вҖ” 4-quadrant grid (work / finance / personal / other) with all open tasks

Click any task title вҶ’ split-view opens with full content, update input, and "done" button.

Write an update like "СҒРҙРІРёРҪРөРј РҪР° РҪРөРҙРөР»СҺ" вҶ’ Claude interprets it, shifts the deadline, and the task title refreshes.

### Edge Functions used

| Endpoint | What it does | Cache |
|---|---|---|
| `format-focus` | Fetches open tasks, sends to Claude for AI titles + dedup + priority ranking | 2h server, 5min client |
| `daily-brief` | Fetches full context (tasks + notes + people + projects), Claude returns `{ greeting, focus, advice, warnings, insight }` | 3h server, 1h client |
| `process-update` | Smart update: Claude parses user text вҶ’ mark done / shift deadline / append content / create follow-up task | вҖ” |
| `open-brain-mcp` | JSON-RPC 2.0 MCP server вҖ” used for `capture_thought` (quick add) and `update_thought` (mark done) | вҖ” |

### Data flow on new tab

1. **Instant render** (< 50ms) вҖ” show cached tasks from `chrome.storage.local`, even if stale
2. **Background refresh** вҖ” fetch `format-focus` + `daily-brief` in parallel, update UI when ready
3. **Stale-while-revalidate** вҖ” user never waits for network on open

### Typography

- **Font family**: system sans (`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto`)
- **Greeting (h1)**: 26px, weight 600
- **Brief focus**: 15px, line-height 1.7, muted
- **Task title**: 18px, line-height 1.5
- **Subtitle labels**: 11px, uppercase, letter-spacing 1px
- **Tags**: 11-12px, pill-shaped, subtle backgrounds

### Layout

- Container max-width: **720px** (focus view), **960px** (with detail panel open), **900px** (all-tasks grid)
- Centered on viewport
- Gear icon fixed top-right, footer fixed bottom-center
- Split layout: task list (left) + detail panel (right, slides in from right with 0.35s ease)

### Existing visual patterns

- **Pills for tags** вҖ” rounded (`border-radius: 12px`), subtle background, small text
- **Expand/collapse** вҖ” details live inside containers with `display: none` вҶ’ `display: block` + `fadeIn` animation
- **Animations** вҖ” `fadeIn`, `slideOut`, `slideIn` for task additions/removals; `transition: flex 0.35s ease` for panel open
- **Hover** вҖ” mostly border/background brightening by ~10%, color shift on text
- **No box-shadows** except on the detail panel (subtle `0 4px 24px rgba(0,0,0,0.3)`)

## Known UX pain points (where design love helps)

1. **The "tower" on wide monitors** вҖ” container is 720px, everything else is empty black space on a 27" display
2. **Density/rhythm** вҖ” task list has 32px margin-bottom between items, looks airy but wastes vertical space; could be tighter
3. **No visual distinction for #1 task** вҖ” all 3 top tasks look the same weight; the "primary focus" could be visually amplified
4. **Brief card when expanded** вҖ” looks like a separate "box" disconnected from the tasks below
5. **Deadline badges** вҖ” 3 colors (green/orange/red), but the green one is barely readable on dark bg
6. **Detail panel** вҖ” opens as a boxed card; could feel more integrated (e.g. backdrop, full-height)
7. **Explore view** вҖ” 4 quadrants, but the headers are tiny and the bucket "РҹСҖРҫСҮРөРө" is an unclear catch-all
8. **Empty states** вҖ” generic Russian text, could be more playful/encouraging
9. **Capture bar** вҖ” feels like an afterthought under the task list; could be more prominent or even inline
10. **Typography hierarchy** вҖ” only 3 sizes really (26 / 18 / 15 / 11-13), could benefit from more dynamic range

## Configuration

Extension needs 3 values (stored in `chrome.storage.local`):

- **Format Focus URL** вҖ” e.g. `https://xxx.supabase.co/functions/v1/format-focus`
- **MCP Endpoint** вҖ” e.g. `https://xxx.supabase.co/functions/v1/open-brain-mcp`
- **API Key** вҖ” matches `MCP_ACCESS_KEY` secret on Supabase

Derived automatically:
- `daily-brief` URL = replace `format-focus` with `daily-brief`
- `process-update` URL = replace `format-focus` with `process-update`

Click the gear icon (top right) to configure.

## Install locally

1. Clone this repo
2. Open `chrome://extensions/` вҶ’ enable **Developer mode** (top right)
3. Click **Load unpacked** вҶ’ select this folder
4. Open a new tab вҶ’ click the gear вҶ’ enter your Supabase endpoints + key
